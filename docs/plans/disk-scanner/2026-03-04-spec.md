# disk-tree: macOS Disk Scanner TUI — Spec

Date: 2026-03-04

## Overview

A Rust TUI application for scanning and freeing up macOS disk space. Displays a file tree sorted by size, highlights macOS-specific cleanup targets, and supports moving items to Trash from within the terminal.

## Architecture

```
disk-tree/
├── src/
│   ├── main.rs          # Entry point, CLI arg parsing
│   ├── scanner.rs       # Parallel directory traversal
│   ├── ui/
│   │   ├── mod.rs       # App state, event loop
│   │   ├── tree.rs      # File tree list widget
│   │   └── statusbar.rs # Bottom bar (disk usage, keybindings, errors)
│   ├── trash.rs         # macOS Trash integration
│   └── highlight.rs     # Special folder detection (caches, brew)
├── Cargo.toml
```

```
│    Documents/                    ███░░░░░  10.2 GB  │
│  ▶ .cache/homebrew   [BREW]      ██░░░░░░   6.8 GB  │

├─────────────────────────────────────────────────────┤
│  ↑↓ Navigate  Enter Open  ← Back  Esc Root  d Trash │
└─────────────────────────────────────────────────────┘
```

| Key           | Action                                      |
|---------------|---------------------------------------------|
| `↑` / `↓`    | Navigate list                               |
| `Enter`       | Open directory (drill down)                 |
| `←` / `Backspace` | Go up one level                        |
| `Esc`         | Jump back to root scan directory            |
| `d`           | Move selected item to macOS Trash (confirm) |
| `q`           | Quit                                        |

- `[CACHE]` / `[BREW]` badges shown on flagged folders
- Progress spinner shown while scanning

## Data Flow

1. **Startup** — parse CLI args (`disk-tree [path]`, defaults to `$HOME`)
2. **Scan** — `walkdir` + `rayon` traverse in parallel, accumulate sizes bottom-up; spinner shown in TUI
3. **State** — in-memory tree of `DirEntry { name, size, children, is_flagged }`, sorted by size descending at each level
4. **Flagging** — after scan, mark nodes matching:
   - `~/Library/Caches/**` → `[CACHE]`
   - `$(brew --cache)` / `~/.cache/homebrew` → `[BREW]`
5. **Navigate** — cursor moves through current dir's children; `Enter` pushes to nav stack, `←`/`Backspace` pops one level, `Esc` resets to root
6. **Delete** — `trash` crate moves item to macOS Trash; parent sizes recalculated up the tree

## Error Handling

| Scenario                     | Behavior                                           |
|------------------------------|----------------------------------------------------|
| Permission denied on dir     | Skip silently; show skipped count in status bar    |
| Trash operation fails        | Show inline error in status bar                    |
| Brew cache detection fails   | Skip BREW badge; degrade gracefully                |

## Testing Strategy

- **Unit tests**: size accumulation logic, flagging heuristics, nav stack behavior
- **Integration tests**: scan a temp directory tree, assert correct sizes and sort order
- **Manual smoke test**: delete flow with macOS Trash verification
- No headless E2E tests (TUI rendering not testable without a terminal)
